## 所有权：Rust 的最大创新

Rust 的所有权系统是它的标志性特性，也是初学者最容易困惑的地方。

### 三个核心规则

1. **每个值都有一个所有者**
2. **同一时间只能有一个所有者**
3. **所有者超出作用域时值被释放**

```rust
fn main() {
    let s1 = String::from("hello");
    let s2 = s1; // s1 的所有权移动到 s2
    
    // println!("{}", s1); // ❌ 编译错误！s1 已失效
    println!("{}", s2); // ✅ 正常输出
}
```

### 借用：不转移所有权地使用

```rust
fn calculate_length(s: &String) -> usize { // s 是对 String 的引用
    s.len()
} // s 离开作用域，但它不拥有该值，不会 drop

fn main() {
    let s1 = String::from("hello");
    let len = calculate_length(&s1);
    println!("'{}' 的长度是 {}", s1, len); // ✅ s1 仍然可用
}
```

## 生命周期：引用的有效期

生命周期标注告诉编译器引用之间的关联关系：

```rust
fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() { x } else { y }
}
```

这个函数签名告诉我们：返回的引用至少与 `x` 和 `y` 中生命周期较短的那个一样长。

## 常见误区

### 误区 1：所有权就是复制

```rust
// 基本类型实现了 Copy trait
let x = 5;
let y = x; // 这里发生了复制，不是移动
println!("{} {}", x, y); // ✅ 正常
```

### 误区 2：引用就是指针

Rust 的引用有编译时检查的借用规则：

- 要么**一个可变引用**，要么**任意数量不可变引用**
- 引用必须始终有效（没有悬垂指针）

## 写在最后

Rust 的所有权系统虽然学习曲线陡峭，但它带来的内存安全保障是无可替代的。一旦理解了设计哲学——**"显式优于隐式，编译时优于运行时"** ——你就会发现，它其实是最贴心的编译器。
